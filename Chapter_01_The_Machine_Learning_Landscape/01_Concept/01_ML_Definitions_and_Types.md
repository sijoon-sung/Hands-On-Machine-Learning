# 01. 머신러닝의 정의와 종류 (Machine Learning Definitions and Types)

## 1. 머신러닝이란? (What is Machine Learning?)

머신러닝은 데이터로부터 학습하도록 컴퓨터를 프로그래밍하는 과학이자 예술이다.

- **일반적 정의:** 명시적으로 프로그래밍하지 않고도 컴퓨터가 학습할 수 있는 능력을 갖게 하는 연구 분야 (Arthur Samuel, 1959).
- **공학적 정의:** 어떤 작업 $T$에 대한 성능 기준 $P$가 경험 $E$를 통해 향상된다면, 이 컴퓨터 프로그램은 학습한다고 말할 수 있다 (Tom Mitchell, 1997).

    - **예시: 스팸 필터**
        - 작업 ($T$): 새로운 메일이 스팸인지 아닌지 구분하는 것
        - 경험 ($E$): 훈련 데이터(유저가 스팸으로 지정한 메일 등)
        - 성능 ($P$): 정확도(올바르게 분류된 메일의 비율)

## 2. 머신러닝의 필요성 (Why Use Machine Learning?)

전통적인 프로그래밍 기법으로는 해결하기 어렵거나 불가능한 문제들을 머신러닝이 해결할 수 있다.

*그림 1. 전통적인 프로그래밍 접근 방법*

![머신러닝 접근법](../../extracted_images/Pasted%20image%2020260128033153.png)
*그림 2. 머신러닝 접근 방법*

1. **복잡한 규칙 대체:** 기존에는 긴 규칙 목록(if-else 등)이 필요했던 문제를 더 짧고 유지보수하기 쉬우며 정확한 코드로 대체 가능하다 (예: 스팸 필터).
2. **유동적인 환경 적응:** 새로운 데이터가 들어오면 자동으로 학습하여 변화에 적응 가능하다 (예: 새로운 스팸 패턴).
3. **복잡한 문제 해결:** 전통적인 방식으로는 해결책이 없는 문제를 해결한다 (예: 음성 인식, 이미지 인식).
4. **통찰력 제공 (데이터 마이닝):** 대용량 데이터를 분석하여 겉으로 드러나지 않던 패턴을 발견하도록 돕는다.

## 3. 머신러닝 시스템의 분류 (Types of Machine Learning Systems)

머신러닝 시스템은 크게 세 가지 기준에 따라 분류할 수 있다.

### 3.1 사람의 감독 하에 훈련하는가? (Supervised vs Unsupervised)

#### **지도 학습 (Supervised Learning)**

알고리즘에 주입하는 훈련 데이터에 **레이블(Label, 정답)**이 포함되어 있다.

- **주요 작업:**
    - **분류 (Classification):** 스팸 필터처럼 클래스를 나누는 것.
      ![분류 예시](../../extracted_images/Pasted%20image%2020260128034634.png)
    - **회귀 (Regression):** 주행거리, 연식 등을 통해 중고차 가격(수치)을 예측하는 것.
      **로지스틱 회귀(Logistic Regression):** 명칭은 회귀이나 주로 **분류**에 사용된다.
      ![회귀 예시](../../extracted_images/Pasted%20image%2020260128033258.png)
- **주요 알고리즘:** k-최근접 이웃(k-NN), 선형 회귀, 로지스틱 회귀, 서포트 벡터 머신(SVM), 결정 트리, 랜덤 포레스트, 신경망 등.

#### **비지도 학습 (Unsupervised Learning)**

훈련 데이터에 레이블이 없으며, 시스템이 스스로 데이터의 구조를 파악해야 한다.
![비지도 학습 예시 1](../../extracted_images/Pasted%20image%2020260128034658.png)
![비지도 학습 예시 2](../../extracted_images/Pasted%20image%2020260128034728.png)

- **주요 작업:**
    - **군집 (Clustering):** 유사한 그룹끼리 묶는 것 (예: 블로그 방문자 그룹화).
    - **시각화 (Visualization) 및 차원 축소 (Dimensionality Reduction):** 데이터의 구조를 시각화하거나, 정보 손실을 최소화하며 데이터를 간소화(특성 추출)하는 것.
    - **이상치 탐지 (Anomaly Detection):** 정상 데이터와 다른 이상 징후 포착 (예: 카드 사기 감지).
    - **연관 규칙 학습 (Association Rule Learning):** 데이터 특성 간의 흥미로운 관계 발견 (예: 기저귀를 산 사람이 맥주도 산다).
- **주요 알고리즘:** k-평균, DBSCAN, PCA, t-SNE, Apriori 등.

#### **준지도 학습 (Semisupervised Learning)**

레이블이 있는 데이터가 적고, 없는 데이터가 많을 때 사용한다 (예: 구글 포토의 인물 태깅 - 소수의 인물만 태깅하면 나머지는 자동 분류).
![준지도 학습 예시](../../extracted_images/Pasted%20image%2020260128045509.png)

#### **강화 학습 (Reinforcement Learning)**
![강화 학습 예시](../../extracted_images/Pasted%20image%2020260128045521.png)
- **에이전트(Agent)**가 **환경(Environment)**을 관찰하고 **행동(Action)**을 실행하며, 그 결과로 **보상(Reward)** 또는 벌점을 받는다.
- 가장 큰 보상을 얻기 위한 최상의 전략(**정책, Policy**)을 스스로 학습한다 (예: 알파고, 보행 로봇).

### 3.2 실시간으로 점진적인 학습이 가능한가? (Batch vs Online)

시스템 자원 활용 및 데이터 처리 방식에 따라 배치 학습과 온라인 학습으로 구분된다.

#### **배치 학습 (Batch Learning)**

- 시스템이 점진적으로 학습할 수 없다.
- 가용한 모든 데이터를 사용하여 오프라인에서 훈련해야 한다.
- 새로운 데이터에 대해 학습하려면 전체 데이터를 사용하여 처음부터 다시 훈련해야 하므로 시간과 컴퓨팅 자원이 많이 소요된다.

#### **온라인 학습 (Online Learning)**
![온라인 학습 구조](../../extracted_images/Pasted%20image%2020260128045717.png)
- 데이터를 순차적으로 한 개씩 또는 **미니배치(Mini-batch)** 단위로 주입하여 시스템을 훈련한다.
- 학습 단계가 빠르고 비용이 적게 들어 데이터가 도착하는 대로 즉시 학습이 가능하다.
- **외부 메모리 학습(Out-of-core learning):** 컴퓨터의 메인 메모리(RAM) 용량을 초과하는 대용량 데이터를 처리할 때 필수적이다. 전체 데이터를 로드하지 않고 일부씩 로드하여 학습하므로, 시스템 메모리 제약을 극복할 수 있는 중요한 기법이다.
- **학습률 (Learning Rate):** 변화하는 데이터에 얼마나 빠르게 적응할지 결정하는 파라미터이다. (너무 높으면 이전 데이터를 빠르게 망각하고, 너무 낮으면 학습 속도가 저하된다).

### 3.3 어떻게 일반화(Generalization)하는가? (Instance-based vs Model-based)

#### **사례 기반 학습 (Instance-based Learning)**
![사례 기반 학습](../../extracted_images/Pasted%20image%2020260128050134.png)
- 훈련 데이터를 단순히 기억(암기)한다.
- 새로운 데이터가 입력되면 기존 데이터와의 **유사도(Similarity)**를 측정하여 분류하거나 예측한다 (예: k-최근접 이웃).

#### **모델 기반 학습 (Model-based Learning)**
![모델 기반 학습](../../extracted_images/Pasted%20image%2020260128050145.png)
- 샘플들의 모델을 생성하여 예측에 사용한다.
- **과정:** 데이터 분석 -> 모델 선택 (예: 선형 모델) -> 훈련 (비용 함수 최소화하여 파라미터 최적화) -> 새로운 데이터 예측(추론).
- **모델 파라미터:** 모델 내부의 설정 값 ($\theta$).
  $life\_satisfaction = \theta_0 + \theta_1 \times GDP\_per\_capita$

## 4. 머신러닝의 주요 도전 과제 (Main Challenges)

머신러닝 프로젝트 실패의 주된 원인은 '부적절한 알고리즘' 또는 '부적절한 데이터'에 기인한다.

### 4.1 나쁜 데이터 (Bad Data)

1. **충분하지 않은 양의 훈련 데이터:** 간단한 문제라도 수천 개의 데이터가 필요하며, 복잡한 문제는 수백만 개가 요구될 수 있다.
   ![데이터 양과 성능의 관계](../../extracted_images/Pasted%20image%2020260128051441.png)
   **데이터의 비합리적 효과성(The Unreasonable Effectiveness of Data):** 성능이 낮은 알고리즘이라도 데이터의 양이 압도적으로 많아지면, 뛰어난 알고리즘보다 더 좋은 성능을 낼 수 있다. 초기에는 알고리즘 간 성능 차이가 크지만, 데이터의 양이 증가할수록 성능이 수렴하는 경향을 보인다.

2. **대표성 없는 훈련 데이터:** 훈련 데이터가 새로운 사례를 적절히 대표하지 못하면 예측 정확도가 저하된다 (예: **샘플링 편향**).
    ![대표성 없는 데이터](../../extracted_images/Pasted%20image%2020260128051632.png)
    점선: 구형 모델(일부 국가 누락)
    대표성이 결여된 데이터로 훈련 시, 일반화 성능이 떨어진 모델이 생성된다.

3. **낮은 품질의 데이터:** 에러, 이상치(Outliers), 잡음(Noise)이 많으면 패턴을 식별하기 어렵다. 데이터 정제(Data Cleaning) 과정이 필수적이다.

4. **관련 없는 특성:** "Garbage In, Garbage Out". 훈련에 유용한 특성을 선택(Feature Selection)하고 추출(Feature Extraction)하는 **특성 공학(Feature Engineering)**이 중요하다.

### 4.2 나쁜 알고리즘 (Bad Algorithm)

1. **과대적합 (Overfitting):**
    ![과대적합 예시](../../extracted_images/Pasted%20image%2020260128051746.png)
    이러한 패턴은 새로운 데이터에 일반화되지 않는다.
    - 모델이 훈련 데이터에는 과도하게 적합되었으나, 일반성이 결여되어 새로운 데이터에 대한 성능이 저하되는 현상이다.
    - **해결:** 모델 단순화, 규제(Regularization) 적용, 훈련 데이터 추가 확보, 잡음 제거.
      ![규제 적용](../../extracted_images/Pasted%20image%2020260128052146.png)
      학습 과정에서 적용할 규제의 양은 하이퍼파라미터로 제어할 수 있다. 하이퍼파라미터는 학습 알고리즘의 파라미터이며, 모델의 파라미터가 아니다.

2. **과소적합 (Underfitting):**
    - 모델이 너무 단순하여 데이터의 내재된 구조를 학습하지 못하는 현상이다.
    - **해결:** 더 강력한 모델 선택, 더 좋은 특성 제공, 모델의 제약(규제) 완화.

## 5. 테스트와 검증 (Testing and Validating)

모델이 새로운 데이터에 얼마나 잘 작동할지(일반화 오차)를 평가하기 위해 데이터를 분할하여 사용한다.

### 5.1 데이터셋의 구분 및 역할

| 데이터셋 | 역할 | 사용 시점 | 주의사항 |
| :--- | :--- | :--- | :--- |
| **훈련 세트 (Training Set)** | 모델 파라미터 학습을 위한 데이터 | 모델 파라미터 학습 | 데이터가 충분히 크고 다양해야 일반화 성능 확보 가능 |
| **검증 세트 (Validation Set)** | 하이퍼파라미터 튜닝 및 모델 선택 | 학습 중간 단계 | 테스트 세트 대신 사용하여 과적합 방지 |
| **테스트 세트 (Test Set)** | 최종 모델의 성능 평가 | 학습 완료 후 | 모델 선택이나 튜닝에 사용 시 데이터 유출 발생 가능 |

- **핵심 포인트:**
    - **훈련 세트**: 모델이 패턴을 학습하는 재료이다.
    - **검증 세트**: 최적의 모델과 하이퍼파라미터를 판단하는 기준이 된다.
    - **테스트 세트**: "이 모델이 실제 환경에서 얼마나 잘 작동할까?"를 확인하는 최종 검증 용도이다.

만약 훈련 오차(Training Error)는 낮지만 일반화 오차(Generalization Error)가 높다면, 이는 모델이 훈련 데이터에 과대적합(Overfitting) 되었음을 의미한다.

### 5.2 하이퍼파라미터 튜닝과 모델 선택

단순히 테스트 세트로만 평가하며 하이퍼파라미터를 조정하면, 모델이 테스트 세트에만 최적화되어 실제 운영 환경에서 성능이 저하될 수 있다. 이를 방지하기 위해 **검증 세트(Validation Set)**를 도입하여 다음과 같은 절차를 따른다.

1. 훈련 세트(검증 세트 제외)로 다양한 모델을 훈련한다.
2. 검증 세트로 최적의 모델과 하이퍼파라미터를 선택한다.
3. 전체 훈련 세트(검증 세트 포함)로 최종 모델을 훈련한다.
4. 테스트 세트로 최종 평가를 수행한다.

- **공짜 점심 없음 (No Free Lunch) 이론:** 데이터에 대한 가정 없이는 어떤 모델이 다른 모델보다 우월하다고 단정할 수 없다. 따라서 여러 모델을 시도하고 검증하는 과정이 필수적이다.

### 5.3 교차 검증 (Cross-validation)

데이터가 부족한 경우 검증 세트를 별도로 분리하면 훈련 데이터가 지나치게 줄어들 수 있다. 이를 해결하기 위해 **교차 검증**을 사용한다. 검증 세트를 여러 개의 폴드(Fold)로 나누어 반복 평가함으로써 더 안정적인 성능 추정이 가능하다. 다만, 계산량이 증가하여 훈련 시간이 길어질 수 있다는 단점이 있다.
