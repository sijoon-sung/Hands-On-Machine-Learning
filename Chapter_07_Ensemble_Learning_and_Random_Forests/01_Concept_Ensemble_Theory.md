## 1. 개요 (Introduction)

**앙상블 학습(Ensemble Learning)**은 여러 개의 예측기(Predictor)를 결합하여 개별 예측기보다 더 좋은 성능을 내는 기법이다. 흔히 **'대중의 지혜(Wisdom of the Crowd)'**라고 부르며, 프로젝트 마지막 단계에서 이미 만들어진 좋은 모델들을 묶어 성능을 극대화할 때 주로 사용한다.

> [!info] **핵심 용어 정리**
> - **앙상블(Ensemble):** 예측기들의 그룹.
> - **앙상블 방법(Ensemble Method):** 앙상블 학습 알고리즘 (예: 랜덤 포레스트).

---

## 2. 투표 기반 분류기 (Voting Classifiers)

### 2.1 직접 투표 (Hard Voting)
- 각 분류기가 예측한 클래스 중 **다수결 투표**로 최종 클래스를 결정하는 방식.
- **강점:** 개별 분류기가 '약한 학습기(Weak Learner)'라도, 충분히 많고 다양하다면 앙상블은 '강한 학습기(Strong Learner)'가 될 수 있음.

> [!abstract] **수학적 배경: 큰 수의 법칙 (Law of Large Numbers)**
> - **동전 던지기 비유:** 앞면 확률이 51%인 동전을 1,000번 던지면 앞면이 다수일 확률은 약 75%, 10,000번 던지면 97% 이상으로 수렴함.
> - 마찬가지로 51% 정확도를 가진 분류기 1,000개를 모으면 앙상블 정확도는 75%까지 기대 가능.
> - **단, 조건:** 분류기들이 서로 **완전히 독립적**이어야 하며, 오차의 상관관계가 없어야 함. (서로 다른 알고리즘을 섞는 이유)

### 2.2 간접 투표 (Soft Voting)
- 모든 분류기가 클래스의 확률을 예측할 수 있다면(`predict_proba()`), 각 확률을 평균 내어 가장 높은 클래스를 선택.
- 신뢰도가 높은 투표에 비중을 더 두기 때문에 직접 투표보다 성능이 좋은 경우가 많음. (SVC는 `probability=True` 설정 필요)

## 3. 배깅과 페이스팅 (Bagging and Pasting)

같은 알고리즘을 사용하되, 훈련 세트의 무작위 부분집합을 달리하여 학습시키는 방법.

### 3.1 개념 구분
- **배깅 (Bagging, Bootstrap Aggregating):** 중복을 허용하여 샘플링 (복원 추출).
- **페이스팅 (Pasting):** 중복을 허용하지 않고 샘플링 (비복원 추출).

### 3.2 편향-분산 트레이드오프
- **🔹 편향(Bias):** 개별 모델은 작은 데이터 조각으로 학습하므로 원래 데이터 전체로 학습했을 때보다 편향은 약간 증가함 (덜 정밀한 규칙).
- **🔹 분산(Variance):** 집계(Aggregation) 과정에서 서로 다른 오류가 상쇄되어 **분산이 크게 감소함**.
- **결과:** 앙상블은 편향은 비슷하게 유지하면서 분산을 줄여 더 안정적이고 일반화 성능이 좋은 모델이 됨.

> [!tip] **장점: 확장성(Scalability)**
> 각 예측기를 독립적으로 **병렬 학습** 시킬 수 있어 여러 CPU 코어나 서버에서 동시 처리가 가능함.

### 3.3 OOB (Out-of-Bag) 평가
- 배깅에서 중복 허용 샘플링 시, 평균적으로 각 예측기는 전체 데이터의 약 **63%**만 사용하고 나머지 **37%**는 사용하지 않음.
- **OOB 인스턴스:** 학습에 사용되지 않은 나머지 37% 데이터.
- **장점:** 별도의 검증 세트 없이 OOB 데이터를 활용해 일반화 성능을 즉시 추정 가능.

## 4. 랜덤 포레스트 (Random Forests)

배깅 방법을 적용한 결정 트리의 앙상블.

### 4.1 핵심 아이디어: 추가적인 랜덤성
- 노드 분할 시 전체 특징 중 최적을 찾는 대신, **무작위로 선택된 일부 특징 중 최적**을 찾음.
- **비유 (시험 문제 풀이):** 
  - **배깅:** 반 전체 학생에게 문제를 풀게 한 뒤 답안을 모아 평균을 냄.
  - **랜덤 포레스트:** 학생들에게 시험 문제의 **일부만** 무작위로 보여주고 답을 쓰게 함. 각자가 다른 문제를 풀기에 다양성이 생기고 전체적으로 튼튼한 결과가 나옴.

### 4.2 엑스트라 트리 (Extra-Trees)
- 특징뿐만 아니라 분할 임계값(Threshold)까지 무작위로 설정하는 방식.
- 최적의 분할 기준을 찾을 필요가 없어 **학습 속도가 랜덤 포레스트보다 훨씬 빠름**.

### 4.3 특성 중요도 (Feature Importance)
- 해당 특징을 사용한 노드가 불순도(Impurity)를 얼마나 줄였는지 측정하여 계산.
- 학습 후 `feature_importances_` 속성으로 확인 가능하며 특성 선택에 매우 유용함.

## 5. 부스팅 (Boosting)

약한 학습기를 순차적으로 연결하여 앞선 모델의 오류를 보정해 나가는 방식.

### 5.1 에이다부스트 (AdaBoost)
이전 모델이 틀린 샘플에 가중치를 높여 다음 모델이 어려운 문제에 더 집중하게 만듦.

> [!formula] **AdaBoost 4단계 학습 로직**
> 1. **에러율($r_j$) 계산:** 모델이 얼마나 중요한 문제(가중치 합)를 틀렸는지 확인.
> 2. **예측기 발언권($\alpha_j$) 결정:** 정확도가 높을수록 큰 발언권을 주고, 무작위 수준이면 0, 그 이하면 음수 가중치 부여.
> 3. **가중치 업데이트:** 똑똑한 모델조차 틀려버린 어려운 문제는 가중치를 기하급수적으로 폭등시킴.
> 4. **최종 예측:** 단순히 다수결이 아니라 각 모델의 발언권($\alpha_j$) 합산 점수로 결정.

### 5.2 그레이디언트 부스팅 (Gradient Boosting)
- 가중치를 수정하는 대신, 이전 예측기가 만든 **잔여 오차(Residual Error)**에 새로운 예측기를 학습시킴.
- **축소(Shrinkage):** 학습률(Learning Rate)을 낮게 설정하여 많은 트리를 사용하게 함으로써 예측 성능을 올리는 기법.

## 6. 스태킹 (Stacking)

- 투표 대신, 모든 예측기의 예측을 취합하는 **블렌더(Blender) 모델**을 별도로 훈련시키는 방법.
- 레이어 1의 예측값들을 입력값(Feature)으로 삼아 레이어 2가 최종 결과를 도출함.
