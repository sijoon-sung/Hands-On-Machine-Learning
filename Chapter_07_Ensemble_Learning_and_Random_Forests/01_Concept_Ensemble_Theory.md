## 1. 개요 (Introduction)

**앙상블 학습(Ensemble Learning)**은 여러 개의 예측기(Predictor)를 결합하여 개별 예측기보다 더 좋은 예측 성능을 얻는 기법이다. 이를 **'대중의 지혜(Wisdom of the Crowd)'**라고도 한다.

- **앙상블(Ensemble):** 예측기들의 그룹.
- **앙상블 방법(Ensemble Method):** 앙상블 학습 알고리즘 (예: 랜덤 포레스트).
- 보통 프로젝트의 마지막 단계에서 이미 만든 좋은 예측기들을 연결하여 성능을 조금 더 끌어올릴 때 사용한다.

> [!abstract] **📘 앙상블 학습(Ensemble Learning) 요약 – 한국어 정리**
> - **지혜의 군중(Wisdom of the Crowd):** 복잡한 질문을 수천 명에게 던지고 답변을 모으면, 종종 전문가의 답변보다 더 나은 결과를 얻을 수 있음. 여러 예측기를 모아 결과를 종합하면, 개별 예측기보다 더 정확한 예측을 얻을 수 있음.
> - **앙상블 학습(Ensemble Learning):** 여러 개의 예측기를 묶어 하나의 강력한 예측기를 만드는 기법. 앙상블 학습 알고리즘을 앙상블 방법(Ensemble method) 이라고 부름.
> - **랜덤 포레스트(Random Forest):** 대표적인 앙상블 방법. 여러 개의 의사결정 트리(Decision Tree)를 서로 다른 무작위 데이터 부분집합으로 학습시킴. 최종 예측 시, 모든 트리의 결과를 모아 다수결 투표로 결정. 단순하지만 매우 강력한 머신러닝 알고리즘.
> - **프로젝트 활용:** 보통 프로젝트 후반부에 앙상블 방법을 사용하여 이미 좋은 예측기들을 결합해 더 나은 성능을 얻음. 머신러닝 대회(예: Netflix Prize)에서 우승 솔루션은 여러 앙상블 방법을 결합한 경우가 많음.

## 2. 투표 기반 분류기 (Voting Classifiers)

정확도가 80%인 여러 개의 분류기(로지스틱 회귀, SVM, 랜덤 포레스트 등)를 훈련시켰다고 가정하자. 더 좋은 분류기를 만드는 가장 간단한 방법은 각 분류기의 예측을 모아 투표하는 것이다.

### 2.1 직접 투표 (Hard Voting)
- 각 분류기가 예측한 클래스 중 **다수결 투표**로 최종 클래스를 결정한다.
- **강점:** 약한 학습기(Weak Learner, 랜덤 추측보다 조금 나은 수준)라도, 충분히 많고 다양하다면 앙상블은 강한 학습기(Strong Learner)가 될 수 있다.
- **수학적 배경 (큰 수의 법칙):**
    - 앞면이 나올 확률이 51%인 동전을 1,000번 던지면 앞면이 다수일 확률은 약 75%가 된다. 10,000번 던지면 97%가 넘는다.
    - 마찬가지로 51% 정확도를 가진 1,000개의 분류기가 있다면 앙상블의 정확도는 75%가 될 수 있다.
- **조건:** 분류기들이 서로 **독립적**이어야 하고, 오차의 상관관계가 없어야 한다. (즉, 서로 다른 알고리즘으로 학습시키는 것이 좋다.)

### 2.2 간접 투표 (Soft Voting)
- 모든 분류기가 클래스의 **확률**을 예측할 수 있다면(`predict_proba()`), 개별 분류기의 예측 확률을 평균 내어 확률이 가장 높은 클래스를 선택한다.
- **성능:** 확률이 높은 투표에 비중을 더 두기 때문에 직접 투표보다 성능이 좋은 경우가 많다.

> [!info] **🗳️ 투표 기반 분류기(Voting Classifier) 요약 – 한국어 정리**
> - **하드 보팅(Hard Voting):** 여러 분류기를 학습시킨 뒤, 각 분류기의 예측 결과를 모아 다수결 투표로 최종 클래스를 결정. 예: 로지스틱 회귀, SVM, 랜덤 포레스트, KNN 등을 결합.
> - **동전 던지기 비유 (Law of Large Numbers):** 편향된 동전(앞면 51%)을 여러 번 던지면 결과가 확률에 수렴. 1,000번 던지면 약 75% 확률로 앞면이 과반수. 10,000번 던지면 97% 이상. 단, 분류기들이 **완전히 독립적**이어야 함.
> - **다양성 확보의 중요성:** 앙상블은 분류기들이 서로 다른 방식으로 학습해야 효과적. 서로 다른 알고리즘을 사용하면 오류 패턴이 달라져 정확도가 향상됨.
> - **소프트 보팅(Soft Voting):** 각 분류기가 예측한 클래스 확률을 평균 내어 가장 높은 확률을 가진 클래스를 선택. 신뢰도가 높은 분류기의 예측에 더 큰 가중치를 주기 때문에 하드 보팅보다 성능이 더 좋음. SVC는 `probability=True` 설정 필요. 실험 결과 소프트 보팅 적용 시 정확도가 91.2% 이상으로 향상됨.

## 3. 배깅과 페이스팅 (Bagging and Pasting)

다양한 분류기를 만드는 또 다른 방법은 같은 알고리즘을 사용하되, 훈련 세트의 서브셋(Subset)을 무작위로 다르게 구성하여 학습시키는 것이다.

### 3.1 개념 구분
- **배깅 (Bagging, Bootstrap Aggregating):** 훈련 세트에서 **중복을 허용하여** 샘플링한다.
- **페이스팅 (Pasting):** 훈련 세트에서 **중복을 허용하지 않고** 샘플링한다.

### 3.2 특징 및 장점
- **병렬 처리 (Parallelizable):** 각 예측기를 독립적으로 병렬 학습시킬 수 있어 확장성(Scalability)이 매우 좋다.
- **편향-분산 트레이드오프:**
    - 개별 예측기는 전체 데이터로 학습한 것보다 편향(Bias)이 높다.
    - 하지만 앙상블을 통해 수집(Aggregation)하면 편향은 비슷하게 유지되면서 **분산(Variance)은 줄어든다**. 결과적으로 과대적합(Overfitting) 위험을 줄여준다.
- 일반적으로 **배깅**이 페이스팅보다 편향은 조금 높지만 분산을 더 잘 줄여주어 선호된다.

> [!abstract] **📘 배깅(Bagging)과 페이스팅(Pasting) 요약 – 한국어 정리**
> - **다양한 분류기 확보 방법:** 같은 알고리즘을 사용하되 훈련 데이터의 무작위 부분집합을 달리함.
> - **배깅:** 중복 허용(with replacement) 샘플링. 같은 인스턴스가 여러 번 선택될 수 있음.
> - **페이스팅:** 중복 불허(without replacement) 샘플링. 같은 인스턴스는 한 번만 선택됨.
> - **특징:** 결과를 집계(Aggregation)하여 최종 예측. 분류는 최빈값(mode), 회귀는 평균값(mean).
> 
> **🔹 편향(Bias)과 분산(Variance)**
> - **편향:** 모델이 얼마나 단순화해서 학습하는지를 나타냄. 데이터를 일부만 사용하면 전체 패턴을 덜 반영하여 편향이 조금 커질 수 있음.
> - **분산:** 데이터의 작은 변화에 얼마나 민감한지를 나타냄. 단일 모델은 과적합 가능성으로 분산이 커짐.
> - **효과:** 개별 모델은 편향이 약간 증가하지만, 집계 과정에서 오류가 상쇄되어 **분산이 크게 줄어듦**. 결과적으로 안정적이고 일반화 성능이 좋은 모델이 됨.

### 3.3 OOB (Out-of-Bag) 평가
- **핵심 아이디어:** 배깅에서는 훈련 데이터를 중복 허용 샘플링하므로, 일부 데이터는 여러 번 선택되고 일부는 아예 선택되지 않음.
- **OOB 샘플:** 선택되지 않은 나머지 훈련 샘플(평균적으로 약 **37%**).
- **장점:** 별도의 검증 세트(Validation Set)를 떼어놓을 필요 없이, OOB 샘플을 사용하여 모델의 일반화 성능을 추정 가능.

### 3.4 특성 샘플링 (Random Patches & Subspaces)
- `BaggingClassifier`는 데이터 인스턴스뿐만 아니라 **특징(feature)**도 무작위로 샘플링 가능. (`max_features`, `bootstrap_features`)
- **Random Patches 방법:** 훈련 샘플과 특징을 모두 무작위로 샘플링. 고차원 데이터(이미지 등)에서 유용.
- **Random Subspaces 방법:** 훈련 샘플은 모두 사용하지만 특징만 무작위로 샘플링. (`bootstrap=False`, `max_samples=1.0`, `max_features < 1.0`)
- **효과:** 특징까지 샘플링하면 예측기 간 다양성이 더 커져 편향은 약간 늘어나지만 분산은 줄어듦.

## 4. 랜덤 포레스트 (Random Forests)

일반적으로 배깅(Bagging) 방법을 적용한 **결정 트리(Decision Tree)의 앙상블**이다.

### 4.1 핵심 아이디어
- 여러 개의 Decision Tree를 모아 만든 앙상블 모델.
- 최종 예측은 다수결(분류) 또는 평균(회귀)으로 합쳐짐.
- **추가 랜덤성:** 노드 분할 시 전체 특징 중 최적을 고르는 대신, **무작위로 선택된 일부 특징 중 최적**을 고름. 트리들이 서로 다른 관점에서 학습하게 되어 다양성이 증가함.

> [!abstract] **Bagging(Bootstrap Aggregating)의 핵심 아이디어**
> - 각 예측기는 훈련 데이터에서 무작위로 샘플링한 부분집합을 사용해 학습. 다양성이 생기고 모델의 분산을 줄임.
> - **방법:** 1. Bootstrap Sampling, 2. 여러 예측기 학습, 3. 결과 결합.
> 
> **쉬운 비유**
> - **Bagging:** 반 전체 학생에게 시험 문제를 풀게 한 뒤, 무작위로 몇 명을 뽑아 답안을 모아 평균을 내는 것.
> - **Random Forest:** 학생들을 뽑을 때, 시험 문제의 **일부만** 무작위로 보여주고 답을 쓰게 하는 것. 각 학생이 다른 문제를 풀어 다양성이 생기고 더 튼튼한 결과를 얻음.

### 4.2 엑스트라 트리 (Extra-Trees)
- **개념:** Random Forest보다 더 무작위성을 가함. 특성뿐만 아니라 **분할 임계값(Threshold)도 무작위로 선택**.
- **장점:** 최적의 임계값을 찾는 시간이 단축되어 학습 속도가 훨씬 빠름. 편향은 커지지만 분산은 더 줄어들 수 있음.

> [!question] **무작위로 선택하는데 왜 분산(variance)이 줄어드는가?**
> 1. **분산이란:** 모델이 훈련 데이터에 따라 결과가 크게 달라지는 정도. 단일 트리는 과적합으로 분산이 큼.
> 2. **다양성:** Random Forest는 다른 데이터 샘플과 다른 특징 조합을 사용. 서로 다른 관점에서 데이터를 바라봄.
> 3. **오류 상쇄:** 여러 트리를 모아 집계하면 개별 트리의 변동성이 서로 보완됨. 무작위성을 추가하면 특정 데이터에 과적합된 트리의 영향력이 줄어듦.
> 4. **비유:** 단일 트리는 한 학생의 실수에 결과가 흔들리지만, Random Forest는 여러 학생에게 문제를 다르게 주어 실수를 서로 보완하게 함.

### 4.3 특성 중요도 (Feature Importance)
- **개념:** 모델 예측 시 어떤 특징이 더 중요한 역할을 했는지 알려주는 지표.
- **계산 방식:** 해당 특징을 사용한 노드가 불순도(impurity)를 얼마나 줄였는지를 기준으로 측정. 모든 트리에서 평균을 내고 합이 1이 되도록 정규화.
- **쉬운 비유:** 시험 문제 풀이 시 어떤 힌트(특징)를 보고 답을 잘 맞추는지 확인하는 것과 같음.

## 5. 부스팅 (Boosting)

약한 학습기를 여러 개 연결하여 강한 학습기를 만드는 방법. 순차적(Sequential)으로 학습하며 앞선 모델의 오류를 보정함.

### 5.1 에이다부스트 (AdaBoost)
- **핵심 아이디어:** 이전 모델이 잘못 분류한 데이터에 더 집중하도록 다음 모델을 학습시킴. 데이터 가중치를 적응적으로 조정함.
- **학습 과정:** 1. 초기화($w=1/m$), 2. 첫 예측기 학습, 3. 가중치 업데이트(틀린 샘플 가중치↑), 4. 새로운 예측기 학습 반복.

> [!formula] **AdaBoost 수식 단계별 해석**
> 1. **모델의 에러율($r_j$):** 전체 가중치 합 대비 잘못 예측한 데이터들의 가중치 합 비율. 단순히 개수가 아니라 '얼마나 중요한 문제'를 틀렸나 측정.
> 2. **모델의 발언권($\alpha_j$):** 정확도가 높으면($r_j \to 0$) 무한대에 가까운 큰 발언권을 주고, 찍는 수준($r_j=0.5$)이면 0을 줌.
> 3. **데이터 가중치 업데이트:** 틀린 데이터는 $\exp(\alpha_j)$를 곱해 증폭시킴. 똑똑한 모델이 틀린 어려운 문제일수록 가중치가 폭등함.
> 4. **최종 예측:** 각 모델이 가진 발언권($\alpha_j$)의 합으로 겨룸. 한 명의 확실한 전문가 의견이 여러 초보자 의견보다 중요할 수 있음.

### 5.2 그레이디언트 부스팅 (Gradient Boosting)
- **작동 원리:** 가중치 수정 대신, 이전 예측기가 만든 **잔여 오차(Residual Error)**에 새로운 예측기를 학습시킴.
- **학습률 (Learning Rate):** 각 트리의 기여도를 조절함. 낮게 설정하면 많은 트리가 필요하지만 성능은 좋아짐(**축소, Shrinkage**).
- **조기 종료 (Early Stopping):** 검증 오차가 향상되지 않으면 훈련을 멈추어 과적합 방지.

## 6. 스태킹 (Stacking)

- **개념:** 예측기의 결과를 취합하는 간단한 함수 대신, **취합하는 모델(Blender / Meta Learner)을 훈련**시키는 방법.
- **작동 구조:** 레이어 1의 예측기들이 값을 예측하면, 그 예측값들을 입력값으로 받아 레이어 2(Blender)가 최종 예측을 수행.
