## 1. 개요 (Introduction)

> [!abstract] **앙상블 학습(Ensemble Learning)이란?**
> 여러 개의 예측기(Predictor)를 결합해서 개별 예측기보다 더 나은 성능을 내는 기법이다. 이걸 **'대중의 지혜(Wisdom of the Crowd)'**라고도 부르는데, 혼자보다는 여러 명이 모이는 게 낫다는 뜻인 듯!
> - **앙상블(Ensemble):** 예측기들의 그룹.
> - **앙상블 방법(Ensemble Method):** 앙상블 학습 알고리즘 (예: 랜덤 포레스트).

- **지혜의 군중:** 복잡한 질문을 수천 명에게 물어보고 답변을 모으면, 가끔 전문가 한 명보다 더 정확할 때가 있다고 한다. 모델도 마찬가지인 것 같다.
- **언제 쓰나?** 보통 프로젝트 마지막 단계에서 이미 만들어둔 괜찮은 예측기들을 합쳐서 성능을 조금이라도 더 끌어올리고 싶을 때 사용한다. 넷플릭스 프라이즈 같은 대회 우승팀들도 앙상블을 엄청 썼다고 들었다.

## 2. 투표 기반 분류기 (Voting Classifiers)

정확도가 한 80% 정도 되는 분류기들이 여러 개 있다면(로지스틱, SVM, 랜덤 포레스트 등), 얘네들끼리 투표를 시켜서 더 좋은 결과를 얻는 방식이다.

### 2.1 직접 투표 (Hard Voting)
- **방식:** 각 분류기가 예측한 값들 중에서 **다수결**로 최종 답을 정한다.
- **신기한 점:** 개별 분류기가 좀 약하더라도(**Weak Learner**), 충분히 많이 모이고 서로 다르면 앙상블 전체는 아주 강력한 **Strong Learner**가 될 수 있다.
- **수학적 근거 (큰 수의 법칙):** 51% 확률인 동전을 1,000번 던지면 앞면이 많을 확률이 75%까지 올라가고, 10,000번 던지면 97%가 넘는다. 모델도 갯수가 많아지면 확실히 유리해지는 것 같다.

### 2.2 간접 투표 (Soft Voting)
- **방식:** 각 분류기가 예측한 **확률**들을 평균 내서 가장 높은 걸 고른다.
- **장점:** "이건 90% 확률로 A야!"라고 확신하는 모델의 의견을 더 잘 반영할 수 있어서, 직접 투표보다 성능이 더 잘 나올 때가 많다. (SVC 쓸 때는 `probability=True`로 해줘야 함!)

> [!tip] **중요한 점: 다양성!**
> 앙상블은 모델들이 서로 **독립적**이어야 효과가 크다. 똑같은 알고리즘만 쓰면 다 같이 틀릴 확률이 높으니까, 서로 다른 알고리즘을 섞어서 오차 패턴을 다르게 만드는 게 핵심인 것 같다.

### 💻 Scikit-Learn 코드 (내가 나중에 보려고 정리)
```python
from sklearn.ensemble import RandomForestClassifier, VotingClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.svm import SVC

# 전문가 3명 소환
log_clf = LogisticRegression()
rnd_clf = RandomForestClassifier()
svm_clf = SVC(probability=True)

# 투표 시작 (소프트 보팅이 성능이 더 잘 나옴!)
voting_clf = VotingClassifier(
    estimators=[('lr', log_clf), ('rf', rnd_clf), ('svc', svm_clf)],
    voting='soft'
)
voting_clf.fit(X_train, y_train)
```

## 3. 배깅과 페이스팅 (Bagging and Pasting)

다양한 알고리즘을 섞는 게 아니라, 똑같은 알고리즘을 쓰되 **데이터를 무작위로 다르게** 나눠줘서 학습시키는 방법이다.

### 3.1 샘플링 방식
- **배깅 (Bagging):** 중복을 허용해서 샘플을 뽑는다 (복원 추출). 똑같은 데이터가 여러 번 들어갈 수 있다.
- **페이스팅 (Pasting):** 중복 없이 샘플을 뽑는다 (비복원 추출).

### 3.2 왜 쓰나? (편향-분산 트레이드오프)
- 개별 모델은 전체 데이터를 다 안 보니까 **편향(Bias)**은 좀 높아질 수 있다.
- 하지만 여러 모델의 결과를 합치면 서로의 실수를 깎아줘서 **분산(Variance)**이 확 줄어든다!
- 결과적으로 과대적합(Overfitting)을 막아주니까 더 안정적인 모델이 된다. (보통 배깅을 더 많이 쓴다고 한다.)

### 3.3 OOB (Out-of-Bag) 평가 (꿀팁!)
- 배깅으로 샘플을 뽑다 보면, 평균적으로 훈련 데이터의 **37%**는 한 번도 선택되지 않는다. 이걸 OOB 샘플이라고 부른다.
- **장점:** 검증 세트(Validation Set)를 따로 안 떼어놔도, 이 OOB 샘플로 모델 성능을 테스트해볼 수 있다. 완전 효율적인 듯!

## 4. 랜덤 포레스트 (Random Forests)

결정 트리(Decision Tree) 여러 개를 배깅 방식으로 묶은 앙상블이다.

### 4.1 핵심: 더 랜덤하게!
- **특성 샘플링:** 노드를 나눌 때 모든 특성을 다 보는 게 아니라, **무작위로 뽑은 몇 개** 중에서만 최선을 찾는다.
- **효과:** 트리들이 서로 더 달라지니까(다양성 증가), 개별 트리는 좀 성능이 떨어져도 앙상블 전체는 훨씬 튼튼해진다.

### 4.2 엑스트라 트리 (Extra-Trees)
- 안 그래도 랜덤한데, 임계값(Threshold)까지 랜덤하게 정해버리는 모델이다.
- **장점:** 어디서 나눌지 고민 안 해도 되니까 학습 속도가 엄청 빠르다! (성능은 랜덤 포레스트랑 비슷비슷한 것 같다.)

### 4.3 특성 중요도 (이거 유용함!)
- 랜덤 포레스트를 돌리고 나면 `feature_importances_`를 통해 어떤 데이터가 중요한지 바로 알 수 있다.
- 꽃잎 길이인지 꽃받침 너비인지, 어떤 게 결정적인지 알려주니까 데이터 분석할 때 꼭 확인해야겠다.

## 5. 부스팅 (Boosting)

약한 모델들을 **순차적(Sequential)**으로 연결해서 앞 모델의 실수를 뒷 모델이 고쳐주는 방식이다.

### 5.1 에이다부스트 (AdaBoost)
앞 모델이 틀린 데이터에 **가중치를 줘서** 다음 모델이 "아, 이건 꼭 맞춰야지!" 하고 집중하게 만든다.

> [!formula] **간단히 이해하는 수식**
> 1. **에러율 ($r_j$):** 모델이 얼마나 중요한 문제를 틀렸나 측정.
> 2. **예측기 가중치 ($\alpha_j$):** 똑똑한 모델일수록 최종 투표에서 발언권을 더 준다.
> 3. **데이터 업데이트:** 틀린 데이터는 다음 모델을 위해 가중치를 확 키운다.

### 5.2 그레이디언트 부스팅 (Gradient Boosting)
가중치 대신, 이전 모델이 만든 **잔여 오차(Residual Error)** 자체를 학습 대상으로 삼는다. (실수를 직접 따라가며 배우는 느낌?)

- **학습률 (Learning Rate):** 이게 낮을수록 성능은 좋아지는데 트리가 많이 필요하다. 적절한 밸런스가 중요한 듯.
- **XGBoost & LightGBM:** 부스팅계의 끝판왕들. 속도도 빠르고 성능도 좋아서 실무에서 거의 이것만 쓰는 것 같다.

## 6. 스태킹 (Stacking)

투표(다수결) 대신, 예측기들의 결과를 다시 입력으로 받아서 최종 결론을 내리는 **새로운 모델(메타 학습기)**을 훈련시키는 방식이다.

1. **1층:** 여러 모델이 각자 결과를 낸다.
2. **2층 (Blender):** 1층 결과를 받아서 "A가 맞네"라고 최종 판단하는 모델을 학습시킨다.
- 좀 복잡해 보이긴 하는데, 성능을 끝까지 쥐어짤 때 쓰는 최후의 수단인 것 같다.
